# トップページ改善メモ

## 目的

トップページは、初めて来た人が「自分はどこを読めばよいか」をすぐ判断できる入口にする。

現在は資料量が多く、法的論点も多いため、トップページでは結論を詰め込みすぎず、立場別・論点別・資料別の入口を明確にする。

## 最優先で見せる入口

1. 保護者向け
   - 入会した覚えがない
   - PTA会費の根拠が分からない
   - 非加入・退会後の扱いが不安

2. PTA役員向け
   - 前年度から引き継いだ運営を直したい
   - 入会申込書・同意書・会費徴収を整理したい
   - 学校との役割分担を明確にしたい

3. 教育委員会・学校向け
   - PTAは任意団体だが、学校の関与は点検対象である
   - 個人情報、会費徴収、施設利用、教職員関与を分けて確認する
   - 学校協力の停止・縮小・条件付けまで含めて整理する

## トップページで強調する軸

- PTA加入は契約であり、入会意思確認が必要
- 学校保有情報をPTA会員管理へ当然に使うことはできない
- 学校徴収金とPTA会費は、請求主体・口座・未納対応を分けて確認する
- 教職員のPTA内部事務は、勤務時間・職務根拠・服務整理を確認する
- 学校施設利用は当然の権利ではなく、目的外使用として条件確認が必要

## 修正方針

- 大型の `index.html` は手作業で全文置換しない。
- 本文変更と JavaScript・検索・サイトマップ更新は別コミットにする。
- トップページ本文は HTML を正本とし、`site.js` で本文を書き換えない。
- カードを増やしすぎず、本文で読ませる。
- スマホでは、最初の1画面で「保護者」「PTA役員」「教育委員会・学校」の3入口が見える構成を目標にする。

## 次にやる具体作業

1. 現在の `index.html` のセクション構成を確認する。
2. 重複している導線や古い説明を洗い出す。
3. 立場別入口の文言を短くする。
4. 主要論点へのリンクを5本程度に絞る。
5. 応援ページへの導線を自然に残す。
6. 修正後に `npm run generate:search` と `npm run generate:sitemap` を実行する。
