# サイト保守メモ

このリポジトリでは、本文HTMLを正とし、JavaScriptは検索、ナビゲーション、地図、開閉などの補助表示に限定する方針で運用します。

## 本文HTMLを正とする範囲

次の内容は、検索・引用・行政説明で使われるため、原則としてHTML本文に置きます。

- PTAの任意加入、公私分離、個人情報、会費徴収、教職員関与、施設利用に関する本文説明
- 法令、行政通知、教育委員会回答、判例、学校文書の読み方
- 教育委員会回答ページの比較基準、参考になる回答例、注意書き
- 主要ページの導入文、結論、確認質問、根拠整理

JavaScriptで後から本文を追加する方式は、検索、引用、ノーJS環境、長期保存の観点で弱くなるため避けます。

## JavaScriptで扱ってよい範囲

- サイト内検索のUI
- ハンバーガーメニューやドロップダウンの開閉
- 地図表示、ピン表示、絞り込みなどの補助的な表示
- `details` 開閉、アンカー位置調整などの操作補助

ただし、JavaScriptで表示する場合でも、重要な説明文そのものはHTML側に残します。

## `board-responses.html` の扱い

`board-responses.html` は、教育委員会回答を引用・比較するための一次資料ページです。

- 「参考になる回答例」のような比較基準はHTML本文に置く。
- 自治体索引、回答本文の開閉、地図、絞り込みはJavaScriptで補助してよい。
- JavaScriptで静的本文と同じブロックを重複生成しない。
- 回答本文のID（例：`#ans-42`）は、外部から引用される前提で不用意に変えない。

## `js/site.js` の扱い

`js/site.js` は全ページ共通の補助処理です。検索、モバイルメニューの開閉、アンカー移動補正、リンク補正などに限定します。

主要ナビゲーションを `innerHTML` で丸ごと差し替える処理は、HTML側の静的ナビゲーションと表示結果がずれ、検索・ノーJS・保守の観点で弱くなるため、段階的に縮小します。既存ページの静的ナビゲーションを整備したうえで、JS側は開閉とアクセシビリティ補助だけに寄せます。

## 生成ファイル

検索インデックスとサイトマップは、次のコマンドで生成します。

```bash
npm run generate:sitemap
npm run generate:search
```

まとめて生成する場合は次を使います。

```bash
npm run generate:all
```

## 生成ファイルの確認

生成結果がコミット済みファイルと一致しているかを確認します。

```bash
npm run check:generated
```

## 自動チェック

Pull Request と `main` への push では、GitHub Actions の `Site checks` が `npm test` を実行します。`npm test` は `npm run check:all` の別名です。

## `site.js` のキャッシュ更新

`js/site.js` を更新した場合、HTML側の読み込み指定も更新します。

```bash
npm run maintenance:bump-site-js -- 89
```

数字は `js/site.js` の版数に合わせて変更します。

## 注意

- 重要本文を JavaScript で差し替えない。
- トップページや主要解説ページの本文は HTML に置く。
- 教育委員会回答ページの重要な比較説明は HTML に置く。
- JavaScriptで同じ静的本文を二重表示させない。
- 重要本文を移した後も、既存アンカーリンクを壊さない。
- `data/site-search-index.js` は検索用の生成ファイルとして扱う。
- 大きなHTMLの置換は、本文変更とキャッシュ更新を分けて行う。
- 一回限りの置換スクリプトは `scripts/legacy/` などへ分け、構造説明Markdownに混在させない。
